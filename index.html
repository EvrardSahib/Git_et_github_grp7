Git est de loin le système de contrôle de version le plus largement utilisé aujourd'hui. Git est un projet open source avancé, qui est activement maintenu.

Durant notre apprentissage de Git et Github, Plusieurs elements imprtant sont à retenir notament les commandes de base tels que:
git init servant à initialiser un projet git,git add . permettant de faire une prise en compte ou simplement un ajour de modifications, git commit -m"description de modification"permettant d'enregistrer un comit ou simplement une modification tout en precisant les differents ajours effectué, git push permettant d'heberger un commit.git branch qui nous a permis de creer des branches secondaires au projet, git merge
Pour la fusion des branches? Il est à noter que la fusion des branches est generalement accompagné de quelques erreurs ou conflits à corriger par le developeur.Une erreur survien generalement lorsque la meme ligne de code a ete modifié par deux developeurs differents. dans ce cas il faut raisoner scrupuleusement pour parvenir à resoudre le probleme. Ce ci represente en quelques lignes le resumé de notre apprentissage sur le versionning. Bien vouloir consulter le support de cours pour plus d'information.
